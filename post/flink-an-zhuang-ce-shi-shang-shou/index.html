flink安装测试上手

2019-07-28

下载

下面是下载地址，从镜像地址下载会比较快，尤其是国内的教育站点： https://flink.apache.org/downloads.html

安装启动

将tgz包放到Mac/Linux设备目录上，我这里有一个旧笔记本安装的CentOS 6.9，所以就直接放在上面的/home/fei/tools目录下了，解压： 然后直接进到bin目录下执行启动脚本即可： 此时可以直接访问http://192.168.0.107:8081，页面会跳转到flink自带的portal页面，可以看到当前有几个可用的slot以及使用情况。

配置说明

登到flink的web页面中我们可以看到默认有一个taskManager和一个slot，这个配置在conf目录下的flink-conf.yaml文件中指定了每个taskManager上的slot个数：

# The number of task slots that each TaskManager offers. Each slot runs one parallel pipeline.

taskmanager.numberOfTaskSlots: 1

conf下的slaves文件指定了taskManager的个数，一个localhost就代表一个taskManager，多写几行localhost就会起多个taskManager：

[fei@fei conf]$ cat slaves
localhost

测试程序

可以到example目录下执行一下测试程序，执行前首先要起一个服务端端口用来监听数据流，模拟不断输入的数据端，可以使用netcat来起端口： nc -l -p 9990 起来后可以使用netstat来查看端口是否有在监听： 启动成功后，则可以在flink目录下执行测试程序，接收端口数据并提交任务到flink： [fei@fei flink-1.7.2]$ ./bin/flink run examples/streaming/SocketWindowWordCount.jar --hostname 192.168.0.107 --port 9990 如果启动报错： 这里有两种可能，一种是9990端口没有正常起监听，按上面说的netstat检查下，或者wget/telnet方式检查下，看是否有响应，另一种可以看flink的日志，在/home/fei/tools/flink-1.7.2/log目录下，看下flink-*-taskexecutor-*.out日志，此日志中可能有如下报错： 注意这里是说MaxDirectMemorySize无效： lnvalid maximum direct memory size: -XX: MaxDirectMemorySize=8388607T

可以将/home/fei/tools/flink-1.7.2/bin目录下的taskmanager.sh的TM_MAX_OFFHEAP_SIZE改为"1024M"，将默认值"8388607T"注释掉作为备份并保存： 此时再重新启动SocketWindowWordCount.jar这个测试程序，然后在起监听的位置输入不同的字符串： log目录下的flink-*-taskexecutor-*.out日志就会打印出统计的单词个数：
